# PR 규칙

## 필수 항목(MUST FOLLOW)

- PR 언어는 한국어를 사용해야한다.
- 반드시 draft PR로 설정한다.
- 임의로 머지 브랜치를 정하지 말고 반드시 유저에게 질의한다.

## PR 양식

아래 내용은 반드시 한국어로 기재한다.

```md
# PR 제목(브랜치 접두사:제목)

## 배경/맥락

## 추가된 기능

## 왜 이렇게 구현했는가

## 수정된 파일

## 테스트 목록(todo)

## 공식 문서

## 관련 PR(옵션)
```

### 브랜치 이름 규칙(MUST FOLLOW)

브랜치 이름에 prefix 를 달아서 사용한다. 의미를 알 수 있는 적절한 이름을 사용한다.
아래의 이름만을 사용한다.

- feat/…
- bug/…
- perf/…
- refactor/…

자잘한 변경점을 모은 브랜치는 **misc** 접두사를 사용한다.
